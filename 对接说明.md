# 后续成员对接说明

本文档说明各通路如何复用已有模块：`gen` 与 `edit` 已实现，成员 3、4 主要对接 hybrid 与总入口。

## 1. 可直接调用的函数

核心函数位于 `reason/gen_pipeline.py`：

```python
from reason.gen_pipeline import run_gen_pipeline

result = run_gen_pipeline(
    prompt="A grass field filled with red poppies and yellow daisies beside a wooden windmill.",
    output_dir="data/output/gen/case0",
    iterations=1,
    dry_run=False,
)
```

参数说明：

- `prompt`: 输入给生成通路的文本 prompt。
- `output_dir`: 当前 case 的输出目录。
- `iterations`: prompt 优化轮数。实验阶段建议 `1` 或 `2`。
- `dry_run`: `True` 时不用 MLLM 优化 prompt；`False` 时调用 `.env` 中的 MLLM。
- `seed`: 可选随机种子，支持的后端会传给出图 API。

## 2. 返回结构

`run_gen_pipeline` 返回 `GenPipelineResult`，可通过 `to_dict()` 转成字典：

```json
{
  "final_image": "data/output/gen/case0/image_iter_1.png",
  "final_prompt": "...",
  "route": "gen",
  "reasoning_chain": [],
  "prompt_before": "...",
  "iterations": [
    {
      "iteration": 0,
      "prompt": "...",
      "analysis": "Baseline generation from the original prompt.",
      "score": null,
      "image_path": "..."
    }
  ],
  "metadata": {
    "dry_run": false,
    "num_iterations": 1,
    "num_candidates": 3
  }
}
```

成员 3 的 `hybrid_pipeline.py` 可以直接复用 `run_gen_pipeline`（见第 5 节）。

## 3. 配置项

复制 `config/.env.example` 为 `config/.env`，再填入自己的密钥。

MLLM 用于 prompt 优化、Reason Agent 与 VQA（推荐 DashScope Qwen-VL）：

```env
MLLM_API_KEY=your_key
MLLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
MLLM_MODEL=qwen-vl-plus
```

T2I 用于真实文生图。当前已支持：

- `T2I_BACKEND=dry_run`: 不调用 API，输出 SVG 占位图。
- `T2I_BACKEND=siliconflow`: OpenAI-like `/images/generations` 接口。
- `T2I_BACKEND=dashscope`: DashScope 原生 Qwen-Image 接口。
- `T2I_BACKEND=comfyui`: 预留本地 ComfyUI 工作流边界。

DashScope Qwen-Image 示例：

```env
T2I_BACKEND=dashscope
T2I_BASE_URL=https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation
T2I_MODEL=qwen-image-2.0
```

EDIT 用于 edit 通路**指令编辑**（可与 T2I 共用密钥）：

- `EDIT_BACKEND=dry_run`: 复制原图或输出 SVG 占位。
- `EDIT_BACKEND=dashscope`: Qwen-Image 多模态接口（**输入图 + 文本指令，无 mask**）。

```env
EDIT_BACKEND=dashscope
EDIT_API_KEY=your_key
EDIT_BASE_URL=https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation
EDIT_MODEL=qwen-image-2.0
```

## 4. edit 通路（成员 2 已实现）

核心函数位于 `reason/edit_pipeline.py`：

```python
from reason.edit_pipeline import run_edit_pipeline

result = run_edit_pipeline(
    image_path="data/input/edit/elephant_squirrel_grass.png",
    instruction="大象和松鼠玩跷跷板会怎样呢?",
    output_dir="data/output/edit/elephant_seesaw",
    iterations=2,
    min_iterations=2,
    candidates=2,
    score_threshold=0.85,
    dry_run=False,
)
```

参数说明：

- `image_path`: 原图路径。
- `instruction`: 假设性/反事实编辑指令（如「如果冰块融化会怎样」）。
- `output_dir`: 当前 case 的输出目录。
- `iterations`: 最大编辑 + VQA 轮数。
- `min_iterations`: 至少跑几轮才允许提前停止（默认 `2`，即至少 refine 一次）。
- `candidates`: 每轮生成的 `edit_prompt` 候选数，逐个出图并 VQA 打分后取最优。
- `score_threshold`: 达到该 VQA 分数且已满足 `min_iterations` 时可提前结束。
- `dry_run`: `True` 时用启发式推理与占位编辑；`False` 时调用 `.env` 中的 MLLM 与编辑 API。

### 4.1 流程概要

参考 [ReasonBrain](https://arxiv.org/abs/2507.01908) 的 HI-IE 任务，本仓库用 **零样本 Agent** 实现类似能力：

```
原图 + 假设性指令
    ↓
Reason Agent（MLLM + prompts/reason_system.txt）
    → reasoning_type / visual_cues / physics_implications / preserve_objects
    → edit_prompt + vqa_checklist
    ↓
finalize_edit_prompt() 注入物理约束与保留对象
    ↓
DashScope Qwen-Image 指令编辑（每轮 N 个候选）
    ↓
MLLM VQA 打分选优（带 reason_context）
    ↓
不合格 → edit_refine / edit_candidate 再编（默认至少 2 轮）
```

**当前默认：指令编辑，不使用 mask inpaint。**

### 4.2 可直接复用的模块

- `reason/reason_agent.py`
  - `run_reason_agent(..., mode="edit")`：结构化推理输出。
  - `finalize_edit_prompt(result)`：组装编辑器可用的完整 prompt。
  - `build_reason_context(result)`：供 VQA / 候选 / refine 使用的摘要文本。
- `reason/edit_client.py` 的 `EditClient`：DashScope Qwen-Image 指令编辑，或 `dry_run` 占位。
- `reason/schemas.py` 的 `ReasonResult` / `EditPipelineResult` / `VQACheck`。
- `reason/edit_verify_loop.py`：薄封装，内部即 `run_edit_pipeline`。

### 4.4 输出目录中的文件

| 文件 | 说明 |
|------|------|
| `image_before.png` | 原图 |
| `image_after.png` | 最终编辑结果（VQA 最优轮） |
| `image_iter_N.png` | 第 N 轮最优图 |
| `reason_analysis.json` | Reason Agent 完整结构 |
| `reason_context.txt` | 推理摘要（VQA/编辑上下文） |
| `reasoning_chain.txt` | 推理链 |
| `edit_final.json` | 完整 JSON 结果 |
| `edit_prompt.txt` | 最终 edit prompt |

### 4.5 `ReasonResult` 约定（edit / hybrid 共用）

```json
{
  "mode": "edit",
  "reasoning_type": "physical",
  "reasoning_chain": ["..."],
  "visual_cues": ["fine-grained facts from the image"],
  "physics_implications": ["expected visible outcome"],
  "target_objects": ["objects or regions to change"],
  "preserve_objects": ["background or objects to keep"],
  "edit_prompt": "...",
  "vqa_checklist": [{"q": "...", "expected": "yes"}]
}
```

`reasoning_type` 四类（与 Reason50K 对齐）：

| 值 | 含义 | 例子 |
|----|------|------|
| `physical` | 物理规律导致的状态变化 | 冰块融化、跷跷板倾斜 |
| `temporal` | 时间维度变化 | 日落、变夜晚 |
| `causal` | 因果/交互事件 | 蜜蜂落在花上 |
| `story` | 隐含叙事/纹理显现 | 隐藏图案出现 |

hybrid 模式将 `edit_prompt` 换为 `scene_prompt`，其余推理字段同样可用。

## 5. 成员 3 对接 hybrid

`reason/reason_agent.py` 已支持 `mode="hybrid"`，输出 `scene_prompt`（与 edit 共用 `prompts/reason_system.txt` 中的推理字段）。成员 3 只需封装 `hybrid_pipeline.py`：

```python
from reason.reason_agent import run_reason_agent
from reason.gen_pipeline import run_gen_pipeline

reason_result = run_reason_agent(image_path, instruction, mode="hybrid")
gen_result = run_gen_pipeline(
    prompt=reason_result.scene_prompt,
    output_dir=output_dir,
    iterations=1,
    dry_run=False,
)
```

这样 hybrid 不需要重新实现文生图，只负责推理目标场景并调用 gen 通路。

## 6. 成员 4 对接 Router / pipeline

统一入口可以按这个约定调用：

```python
if route == "gen":
    return run_gen_pipeline(prompt, output_dir).to_dict()
elif route == "edit":
    return run_edit_pipeline(image_path, prompt, output_dir).to_dict()
elif route == "hybrid":
    return run_hybrid_pipeline(image_path, prompt, output_dir)
```

Router 分流建议（有图 + 假设性指令时）：

| 变化类型 | 通路 |
|----------|------|
| 局部物理/状态变化（融化、变色、倾斜） | `edit` |
| 整图构图/数量/氛围大变（分束花、昼夜切换） | `hybrid` |

`gen` 通路的结果已经包含 `route="gen"`，方便总入口统一展示。
